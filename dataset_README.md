# asia-fertility — results

Tokenizer fertility, cost ratios, and multi-turn needle-in-haystack recall measurements for 16 lower-resource Asian languages × 9 frontier tokenizers, with bootstrap 95% confidence intervals.

Generated by [asia-fertility](https://github.com/Helmo21/asia-fertility) v0.2.0.

## Configs (use with `datasets`)

```python
from datasets import load_dataset

# Main leaderboard: 1 row per (language × tokenizer × corpus) cell
ds = load_dataset("Helmo21/asia-fertility", "leaderboard")

# NIAH benchmark: 1 row per (model × language × fill × position × trial) cell
niah = load_dataset("Helmo21/asia-fertility", "niah")
```

## Files per version (under `v{version}/`)

| File | Description |
|---|---|
| `results.csv` | Main leaderboard — 144–160 rows × `lang, tokenizer, fertility, premium, cost_ratio, cpt, bpt, ...` |
| `results.json` | Same as `results.csv` in JSON list form |
| `leaderboard.json` | Nested schema-versioned view (consumed by the [web app](https://fertiscope.vercel.app)) |
| `manifest.json` | Run metadata: SHA256 of config + pinned prices + FX rates + tokenizer versions |
| `niah_results.csv` | Needle-in-haystack benchmark (~536 rows) — 1 row per (model, lang, fill, position, trial) |

## Columns in `results.csv` (the main leaderboard)

- **Identity**: `corpus`, `lang`, `iso`, `script`, `family`, `tokenizer`, `tokenizer_family`, `tokenizer_backend`
- **Sums**: `n_sentences`, `tokens_sum`, `words_sum`, `chars_sum`, `bytes_sum`
- **Metrics (with 95% bootstrap CIs)**: `fertility`, `fertility_ci_low`, `fertility_ci_high`, `premium`, `premium_ci_low`, `premium_ci_high`, `cost_ratio`, `cost_ratio_ci_low`, `cost_ratio_ci_high`, `cpt`, `cpt_ci_low`, `cpt_ci_high`, `bpt`, `bpt_ci_low`, `bpt_ci_high`
- **Context efficiency** (effective chars per window): `ctx_eff_4096`, `ctx_eff_8192`, `ctx_eff_32768`, `ctx_eff_131072`
- **Provenance**: `segmenter_used`, `tokenizer_unavailable`, `skip_reason`

## Columns in `niah_results.csv`

`model, lang, iso, script, fill_tokens, position_pct, trial, marker, actual_tokens, marker_position_tokens, response, recalled, error, elapsed_seconds`

## Key findings (v0.2.0)

- **Cost ratios reproduce** the FertiScope hackathon paper Table 1 within bootstrap CIs (Tamil cl100k 7.61× vs paper 7.19×; Burmese 11.66× vs 11.20×).
- **Gemma-2** is the best open-weight tokenizer for South-Asian deployment (Tamil 2.58×, Burmese 4.80×).
- **BLOOM** has surprisingly low cost ratio on Indic (Tamil 1.29×, Bengali 1.17×).
- **NIAH recall collapses** to 0–7% on Hindi/Tamil/Burmese/Lao with script-native markers, *even at 4k context*, across gpt-4o-mini, gpt-3.5-turbo, and llama-3.1-8b-instruct — a finding the original FertiScope hackathon paper missed because it used Latin markers.
